What happens when you initialize a repository? Why do you need to do it?

    It creates files and folders and allows you to start saving versions of you files. 

How is the staging area different from the working directory and the repository? What value do you think it offers?

    The staging area is an area that you want to save a version of your file before sending them to the repository. The working
    directory is the area on your computer that you are working on the file. This allows you to send multiple files part of your
    version at once and gives you the abilty to have a way to verify before comitting them to the repository.

How can you use the staging area to make sure you have one commit per logical change?

    Use git diff to verify the files are staged correctly before comitting.
